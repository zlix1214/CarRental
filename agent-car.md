# Codex 協作與重構規範

## 1. 撰寫風格與程式碼品質

本專案目前進行架構重構，Codex 在改寫程式碼時必須嚴格遵守以下品質規範：

- 簡潔與可讀性：程式碼應保持乾淨，避免過度封裝。多數元件應保持單一職責，並優先提取重複邏輯。
- 命名嚴謹度：變數、函式與元件命名必須具備明確的業務語意，禁止使用模糊的縮寫。

## 2. 輸出回報格式規範

Codex 在每次完成工作或提出企劃時，回覆必須嚴格遵守以下結構，不得包含任何無意義的裝飾性符號（如 Emoji）：

### [本次改動企劃/成果]

- 精確說明本次具體修改了哪幾個檔案、動了哪些邏輯。

### [剩餘進度與下一動預告]

- 標註目前整體重構的預估進度，以及下一動建議修改哪一個檔案或元件。

### [技術細節與觀念抽出]

- 將本次改動中涉及的核心技術與底層觀念單獨抽點條列。
- 此部分必須將「代碼實現」與「核心觀念（如快取機制、型別定義、異步處理原理）」切開
